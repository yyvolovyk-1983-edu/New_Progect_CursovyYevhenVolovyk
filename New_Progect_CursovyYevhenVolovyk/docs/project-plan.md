# План курсового проєкту

## 1. Тема і предмет

- Тема: Стабілізація каналу крену квадрокоптера Х-схеми.
- Предмет: Моделювання та синтез систем автоматичного управління.
- Курс: ____________
- Викладач: ____________

## 2. Мета

Розробити та дослідити систему автоматичного управління каналом крену квадрокоптера з використанням лінійної моделі об'єкта, станового опису та PID-регулятора.

## 3. Завдання

1. Проаналізувати об’єкт управління та визначити фізичні параметри каналу крену.
2. Побудувати модель об'єкта у просторі станів для кута крену.
3. Визначити вплив шуму вимірювань і зовнішніх збурень.
4. Синтезувати PID-регулятор для стабілізації каналу крену.
5. Додати опис блоку насичення та обмежень виконавчого механізму.
6. Провести імітаційне моделювання та проаналізувати результати.
7. Оформити пояснювальну записку відповідно до вимог викладача.

## 4. Методологія

- Методи: математичне моделювання, простір станів, PID-контроль, імітаційне моделювання.
- Інструменти: Markdown, Python, графічні ілюстрації, текстовий редактор.
- Мови/платформи: Python для симуляції, стандартні офісні інструменти для оформлення документації.

## 5. Етапи виконання

1. Підготовка
   - Збір вимог з `docs/Вимоги_до_КП/`.
   - Огляд наявних матеріалів у `docs/матеріали для роботи/`.
   - Визначення системи координат і позначень.
2. Проектування
   - Формування схем і блокових діаграм.
   - Побудова рівнянь руху та станового опису.
   - Визначення структури звіту згідно з шаблоном викладача.
3. Реалізація
   - Розрахунок параметрів моделі.
   - Синтез PID-регулятора.
   - Розробка симулятора в `src/simulation.py`.
4. Тестування
   - Аналіз перехідної характеристики.
   - Перевірка впливу збурень і шуму.
   - Порівняння поведінки з/без обмежень.
5. Оформлення звіту
   - Заповнення розділів записки.
   - Перевірка оформлення, формул і ілюстрацій.
   - Підготовка PDF/ODT-файлів.

## 7. Ризики

- Невизначені параметри квадрокоптера (J_x, K, габарити).
- Невідповідність формул і рисунків одній системі координат.
- Наявність шуму та збурень, що знижує точність моделі.
- Нестача сучасних джерел 2020+ у літературі.
- Порушення формату оформлення пояснювальної записки.

## 8. Критерії успіху

- Повний опис об’єкта та основних змінних.
- Коректна модель у просторі станів з визначеними матрицями.
- Наявність аналізу шуму і обмежень актуатора.
- Реалізована проста симуляція для перевірки управління.
- Оформлений звіт, що відповідає вимогам викладача.
- Наявність контрольного списку для верифікації готовності.

## 9. Обмеження проєкту

- Використовувати одну єдину систему координат для каналу крену.
- Розглядати малий кут крену та лінійну модель.
- Модель має бути другого порядку за передаточною функцією.
- У моделі врахувати шуми вимірювань і зовнішні збурення.
- Моделювання має бути зроблено в `src/simulation.py` з базовим PID-контролем.
