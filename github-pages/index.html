01

AI Agent Suite for Aerospace Academic Writing

Claude Code Anthropic API Aerospace Production

A Ukrainian aerospace university had no tooling for thesis and coursework document generation. Students spent 30–40 hours per project on formatting, citations, and ДСТУ standard compliance — time better spent on engineering work.

I built a suite of 5 interconnected Claude Code agents: a generator that creates full document structure from an assignment brief, a reviewer that audits against ДСТУ 3008:2015 formatting requirements, a corrector that applies professor feedback precisely, and two domain experts for avionics standards (DO-178C, ARP4754A, ARINC 429) and thesis formatting.

Result: Document preparation time reduced from ~40 hours to ~8 hours. Agents actively verify compliance with aviation certification standards in real time.
02

Multi-AI Orchestration: Claude + Gemini + Groq

Claude Code Gemini 2.5 Pro Groq Open Source

Single-model AI assistants have blind spots. Claude is deep on reasoning and code, Gemini is better at factual cross-checks, Groq excels at fast structured outputs. Using them separately is slow; using them together requires infrastructure.

I designed and deployed a production multi-AI pipeline under Claude Code: automatic task routing to the optimal model, cross-model consensus for critical decisions, independent "second opinion" verification via Gemini 2.5 Pro, and a health-monitoring system for the entire infrastructure. Full Ukrainian-language support throughout.

Result: 10/10 components operational. Verification time for critical decisions reduced by ~60% via parallel model querying. 4 reusable skills deployed.
03

Smart Home Security Audit — 23 Findings Including CVE-2015-0552

Security Audit CVE Nmap Wireshark 8-Phase Methodology

Conducted a full 8-phase security audit of a residential smart home environment: two routers on separate subnets, one Windows workstation, IoT devices, cloud services, and browsers. The audit uncovered a critical published CVE that the device manufacturer has never patched — the product has been EOL for years but remains in active use in millions of homes.

Critical findings: UDP backdoor port (CVE-2015-0552) present on primary router firmware; EOL web server (Boa HTTPd 0.94.14rc21, no patches since 2013) serving the admin interface; SSLv2/SSLv3 enabled with expired certificate issued in 2013.

High findings: POODLE (CVE-2014-3566), SWEET32 (CVE-2016-2183), RC4 cipher suites, HTTP Digest realm leaking device model, a second undiscovered router subnet, LuCI admin tokens stored in browser history.

Result: Full 40-page report with severity ratings, reproduction steps, and remediation guidance. External IP confirmed not indexed by Shodan (no public exposure at time of audit). Remediation table created for all 23 findings.
